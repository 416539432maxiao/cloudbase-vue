## 一、产品概述

### 1.1 产品简介

Coolteacher 是一款面向中小学教师的**游戏化班级积分管理工具**。通过"宠物养成 + 积分激励"的双重机制，帮助教师高效管理班级日常行为表现，激发学生的学习积极性和竞争意识。

**核心逻辑**：学生表现好 → 老师加分 → 宠物成长进化 → 收集更多宠物 → 积分兑换奖励 → 持续正向激励循环。

### 1.2 目标用户

- **主要用户**：中小学班主任、科任教师
- **间接用户**：学生（通过投屏查看宠物和积分）；家长（通过分享链接查看）

### 1.3 产品定位

| 维度 | 说明 |
|------|------|
| 平台 | Web 应用（支持 PWA 安装到桌面） |
| 设备支持 | 电脑、平板、手机、希沃一体机等 |
| 前置条件 | 已有登录系统（本模块基于已有用户体系） |

### 1.4 与已有系统的关系

- ✅ **已完成**：用户注册 / 登录功能
- 🆕 **本次新增**：班级积分管理模块（本文档范围）

### 2.2 成长积分体系

Coolteacher 采用**成长积分**同时驱动"宠物成长"和"累计排名"：

| 积分类型 | 用途 | 变化规则 |
|---------|------|---------|
| **成长积分** | 喂养宠物、升级进化 | 加分时 +N，扣分时 -N（最低为 0，不可为负） |
| **累计成长积分** | 光荣榜排名依据 | 加分时 +N，扣分时 -N |

**计算逻辑**：

- **加分操作**：成长积分 +N，累计成长积分 +N（根据不同任务增加不同积分）
- **扣分操作**：成长积分 -N，累计成长积分 -N（成长积分最低为 0，不能为负数）
- **领养新宠物**：仅重置成长积分为 0

> [!IMPORTANT]
> 扣除成长积分**会导致宠物降级**。当成长积分降低至前一等级的阈值以下时，宠物会自动回退到对应等级。

### 2.3 宠物养成机制

每个学生可以领养一只宠物，通过获得成长积分驱动宠物成长进化。

#### 2.3.1 进化阶段

每只宠物有 **5 个成长阶段**，随成长积分提升自动进化：

| 等级 | 形态 | 说明 |
|------|------|------|
| Lv.1 | 🥚 蛋 | 刚领养时的初始形态 |
| Lv.2 | 👶 幼年期 | 破壳而出，萌萌的幼崽 |
| Lv.3 | 🧒 少年期 | 开始成长，初具特色 |
| Lv.4 | 💪 成年期 | 威风凛凛的成熟形态 |
| Lv.5 | 👑 究极体 | 最终进化，满级收集！ |

#### 2.3.2 成长阈值配置

教师可自定义 Lv.1 到 Lv.5 每个等级所需的成长值。

**默认值**：

| Lv.1 | Lv.2 | Lv.3 | Lv.4 | Lv.5 |
|------|------|------|------|------|
| 0 | 5 | 15 | 30 | 50 |

教师可根据班级情况调整进化节奏（如缩短/延长升级所需时间）。

#### 2.3.3 宠物领养与更换

**领养方式**：

| 方式 | 说明 |
|------|------|
| 学生自选 | 投屏班级主页，学生自己点击卡片上的「领养」按钮 |
| 系统随机分配 | 点击「一键随机分配」，系统为全班随机分配；如果之前学生已经领养了宠物，设置为随机分配后，所有宠物都会被随机替换为新宠物 |

**更换规则**：

- 喂养期间**不可以**更换宠物
- 满级（Lv.5）后可领养新宠物，成长值重置为 0
- 可以查看每个学生历史领取过哪些宠物

#### 2.3.4 满级收集机制

当学生的宠物达到 **Lv.5 满级**时：

1. 🎉 屏幕出现进化庆祝动画
2. 当前宠物自动加入「已收集」列表，获得一枚**徽章**
3. 学生可以领养一只新宠物，成长值重置为 0
4. 收集越多宠物，徽章越多，排名越靠前
5. 学生卡片上展示已收集的徽章

### 2.4 积分规则管理

教师可自定义加分 / 扣分规则：

| 属性 | 说明 |
|------|------|
| 规则名称 | 如"作业优秀"、"课堂纪律差" |
| 分值 | 正数为加分，负数为扣分 |
| 图标 | 可选 emoji 或系统图标 |
| 排序 | 支持拖拽排序 |

**系统预设规则**（教师可修改 / 删除 / 新增）：

**加分项**：

| 规则 | 分值 |
|------|------|
| 🌟 课堂表现优秀 | +3 |
| 📝 作业优秀 | +2 |
| 🙋 积极举手 | +2 |
| 🤝 帮助同学 | +1 |
| 📖 早读表现好 | +1 |
| 🏆 考试进步 | +5 |
| 🤝 值日认证 | +1 |
| 🤝 答对问题| +2 |

**扣分项**：

| 规则 | 分值 |
|------|------|
| 📵 上课说话 | -3 |
| 📋 未交作业 | -2 |
| ⏰ 迟到 | -1 |
| ⚠️ 打瞌睡 | -2 |

## 三、功能需求详述 (Functional Requirements)

> 本节按教师的**使用旅程**组织模块：权限授权 → 基础配置 → 核心互动 → 激励反馈 → 数据复盘。

### 3.1 权限与授权模块

#### 3.1.1 激活流程与校验规则

Coolteacher 采用**激活码**作为付费/授权入口。用户初次使用积分管理模块时才显示激活码入口，也可在线付款激活。

**激活流程**：

1. 用户完成注册/登录后，初次使用积分管理模块时提示激活
2. 用户将获得的 **6 位激活码**（字母/数字，不区分大小写）粘贴到输入框
3. 点击「立即激活」
4. 激活成功后解锁积分管理系统的全部功能

**激活码规则**：

| 规则 | 说明 |
|------|------|
| 格式 | 6 位字母/数字混合，不区分大小写 |
| 使用次数 | 每个激活码仅能使用一次 |
| 有效期 | 永久版 |

### 3.2 基础配置模块（准备工作）

#### 3.2.1 班级管理

**创建班级**：

- 点击「新建班级」，输入班级名称（如"三年三班"）
- 支持「默认模板」或「从已有班级复制设置」两种创建方式
- 创建班级时需要选择**宠物图鉴风格**
- 创建班级时需要选择**领养方式**：学生自主选宠物 或 系统随机分配宠物
- 每个老师最多创建 **15个班级**

**班级操作**：

- 顶部导航栏提供班级快速切换入口
- 班级可以重命名
- 班级可以删除；删除班级会删除数据库中该班级的所有数据，因此删除时需要**再次输入班级名称**才能确认删除

**班级数据隔离原则**：


> 每个班级的数据**完全独立**，互不影响。切换班级相当于切换到一个完全独立的工作空间。

以下数据均为班级级别独立：

| 数据 | 说明 |
|------|------|
| 学生名单 | 每个班级有自己的学生列表 |
| 宠物分配 | 每个学生的宠物绑定在所属班级内 |
| 积分数据 | 成长积分为班级内独立 |
| 积分规则 | 每个班级有独立的积分规则；新建班级时复制已有班级的规则，修改不影响原班级 |
| 商城商品 | 每个班级有独立的商品列表和库存 |
| 成长记录 | 加减分明细、统计数据按班级隔离 |
| 分组设置 | 小组划分按班级独立管理 |
| 光荣榜 | 排名仅在班级内部计算 |

#### 3.2.2 学生与分组管理

**添加学生**（支持两种方式）：

| 方式 | 操作 |
|------|------|
| 单个添加 | 输入姓名，点击「添加」 |
| 批量添加 | 文本框一行一个名字，支持直接复制 Excel 表格 |

- 班级的学生名单支持增删改

**分组管理**（支持将学生分组进行小组 PK）：

- 创建小组，设置名称和图标；可以用不同颜色的旗子标识，支持 **20 种颜色**的旗子
- 将学生分配到小组
- 小组积分 = 成员累计积分之和
- 支持全组统一加分或减分

#### 3.2.3 积分规则配置

参见 [2.4 积分规则管理](#24-积分规则管理)。教师可在「教师设置 → 成长与积分」中自定义加扣分规则的名称、分值、图标，并拖拽排序。

### 3.3 核心互动模块（高频操作）

#### 3.3.1 班级主页

班级主页是教师日常使用频率最高的页面，所有增加成长积分和扣积分的操作在此完成。

**学生卡片**：每个学生以卡片形式展示：

| 信息 | 说明 |
|------|------|
| 姓名 | 学生名称 |
| 等级 | Lv.1 ~ Lv.5 |
| 宠物图片 | 当前宠物及形态名（如「暗影豹 - 幼年形态」） |
| 成长进度条 | 当前成长值及进度 |
| 已收集徽章 | 满级后出现，显示已收集的徽章数 |

**视图模式**：

| 模式 | 说明 |
|------|------|
| 默认模式 | 学生卡片常规展示，含宠物图片 |
| 迷你模式 | 一屏展示全班概况 |
| 小组模式 | 按小组分组展示，支持全组加分 |

**搜索与排序**：

- 搜索：输入学生姓名快速定位
- 排序：支持「默认排序」「按姓名」「按积分」三种

**锁屏功能**：

- 投屏时可锁定屏幕，防止学生误操作
- 支持设置/修改锁屏密码

#### 3.3.2 宠物状态展示与交互

**学生加入初始状态（Egg State）**：

- 新加入的学生卡片显示一只"宠物蛋"
- 图片：发光的宠物蛋
- 内容区：显示"点击选择你的第一个伙伴"引导语

**宠物选择弹窗（Pet Selection Popup）**：

- 触发机制：点击"宠物蛋"卡片
- 标题：选择你的第一个伙伴
- 展示区：展示神兽图鉴中的各类宠物（当前班级所选风格）
- 交互：点击宠物卡片进行选择，点击确认按钮完成

#### 3.3.3 打分操作（喂养宠物）

**单个学生打分**：

1. 点击学生卡片
2. 弹出「加分喂养」面板，展示所有积分规则；每条规则以独立卡片展示，默认规则内置符合场景的 SVG 图标，用户自定义规则使用统一标识 SVG
3. 选择「加分」或「扣分」卡片
4. 点击对应规则（如「作业优秀 +3」），立即生效，并播放音效（加分与扣分音效不同）

**批量打分**：

1. 点击「批量模式」按钮
2. 勾选多个学生（支持搜索框输入多个姓名，空格分隔）
3. 底部显示已选人数
4. 点击「批量喂养」，选择规则统一加分/扣分
5. 可以对小组进行批量打分：勾选某些人，也可以勾选整个小组

**全班操作**：

- 点击「全班操作」→ 选择规则 → 全班统一加分/扣分（如早读打卡 +1）

**撤回机制**：

- 操作后 **8 秒内**底部弹出撤回条，点击「撤回」可取消
- 超时后可在「成长记录」页面撤回

### 3.4 激励与反馈模块

#### 3.4.1 积分商城（小卖部）

学生用**徽章**兑换奖励，让成长积分兑换的徽章真正有价值。

**兑换流程**：

1. 教师在积分商城页面查看商品列表
2. 点击商品「兑换」按钮
3. 选择要兑换的学生
4. 系统自动检查学生徽章数是否足够
5. 确认后扣减学生徽章数，库存 -1

**商品管理**：每个商品包含：

| 属性 | 说明 |
|------|------|
| 名称 | 商品名称 |
| 图标 | 自己设计好的3D精美图标 |
| 描述 | 商品简介（可选） |
| 价格 | 所需徽章数 |
| 库存 | 可兑换数量 |

支持新增、编辑、删除、拖拽排序，修改自动保存。

**系统预设商品**：

| 商品 | 价格 |
|------|------|
| 铅笔 | 10 分 |
| 作业本 | 15 分 |
| 小零食 | 20 分 |
|  前排座位券 | 30 分 |
|  选同桌券 | 40 分 |
| 小组长体验 | 40 分 |
| 免作业卡 | 50 分 |
|  当一天班长 | 60 分 |

每个班级的商品库存独立管理。

**兑换记录**：

- 查看历史兑换记录
- 包含：学生姓名、商品名称、兑换时间、扣除徽章数


- 增加趣味性和期待感

#### 3.4.2 光荣榜

展示全班学生积分排名，激励良性竞争。

> **设计说明**：成长积分（XP）是成长与排名的唯一依据；徽章是累计到一定积分后解锁的奖励，专用于商城兑换商品，两者功能分离，不混用。

**个人榜**：

- 按**累计成长积分**（XP）高低排序
- 前三名用 🥇🥈🥉 金银铜色高亮
- 每位学生显示：等级、持有徽章数、累计成长积分
- 支持按「总榜」「周榜」「月榜」查看

**小组 PK 榜**：

- 按小组**总成长积分**排名
- 显示每组人数、总积分、人均积分
- 可展开查看小组成员详情

#### 3.4.3 宠物图鉴 (Pet Pokedex)

作为系统的成就收集中心，展示所有可用的宠物包及其各阶段形态。

**核心功能**：

| 功能 | 说明 |
|------|------|
| **分类浏览** | 按美术风格（可爱、像素、写实等）分类展示不同的宠物图包 |
| **进化全预览** | 老师可以点击任一宠物，查看其从 Lv.1 🥚 到 Lv.5 👑 的完整进化轨迹图 |
| **背景故事** | 展示每种宠物的简短介绍和进化说明（增加趣味性） |
| **图鉴达成度** | 展示当前班级已解锁（满级）的宠物种类占比 |

**交互说明**：
- 在图鉴中，所有形态均为可见状态（预览模式），用于激励学生努力获得积分以解锁这些形态。
- 支持切换宠物风格预览，帮助老师决定班级要采用哪种视觉包。

### 3.5 数据与家校模块

#### 3.5.1 成长记录（数据中心）

全面的班级积分数据分析。

**明细记录**：

| 字段 | 说明 |
|------|------|
| 学生姓名 | 操作对象 |
| 规则名称 | 加分/扣分规则 |
| 分值 | +/- 具体分数 |
| 操作时间 | 精确到分钟 |


- 按时间倒序展示
- 支持按学生姓名搜索、按日期筛选、按加分/扣分筛选
- 支持撤回未撤回的记录



#### 3.5.2 家长分享

教师可生成分享链接，让家长实时查看孩子的宠物成长和积分情况。

**分享方式**：

| 类型 | 说明 |
|------|------|
| 班级分享 | 全班学生的宠物卡片、等级、进度，前三名带勋章 |
| 学生分享 | 单个学生的详细信息 |

**家长查看内容**：

- **班级分享**：全班学生宠物卡片、等级、进度排名，前三名带金银铜勋章
- **学生详情**：宠物大图和等级进度、徽章数、累计积分、已收集的宠物列表、最近 10 条积分记录

> 📝 分享链接**无需登录**即可查看，方便家长随时了解。

### 3.6 统计分析大盘 (Data Dashboard)

为教师提供可视化的班级积分数据复盘，支持多维度分析班级表现。

**核心展示维度**：

| 维度 | 展示形式 | 说明 |
|------|------|------|
| **关键指标 (KPI)** | 统计卡片 | 累计总积分、本周加分次数、全班平均分、最活跃学生 |
| **趋势分析** | 折线图 | 展示班级积分在最近 7 天 / 30 天的波动趋势 |
| **规则触发分布** | 环形图 / 饼图 | 分析哪些加分项或扣分项被触发最频繁，帮助老师了解班级共性行为 |
| **规则热门分布** | 水平柱状图 | 展示学生在不同加分维度上的表现分布 |

**交互要求**：
- 支持按时间段（今日、本周、本月、自定义）筛选数据。
- 支持切换班级查看各班级对比（横向复盘）。

## 四、交互与视觉规范 (UI/UX Guidelines)

> 本节将前端展示与交互细节集中整理，专供 UI/UX 设计师和前端开发参考。
> 详细的色彩参数、阴影数值、动效曲线等请参见 [`design.md`](file:///d:/uni/coolpoints/design.md)。

### 4.1 整体视觉风格

采用 **Claymorphism（粘土立体风）** 视觉处理，结合轻度拟物化：

1. **深度阴影（Deep Shadows）**：卡片外层采用带主色调的深层柔和发光厚阴影（`Y偏移 12~16px，模糊半径 24~32px，颜色 10% 主色调`），让卡片看起来漂浮在物理桌面之上。
2. **边缘高光（Inner Strokes）**：通过 2px 的白色或柔化高亮 Inner Stroke 模拟卡片边缘被光照亮的倒角感。
3. **实体按钮效果（Hard Button Shadows）**：关键徽章（如 Lv 角标、SVG 追踪器节点）采用硬底阴影（`Y偏移 2~3px，模糊半径 0px`），模拟可按压的物理实体按钮质感。
4. **雕刻化内阴影（Inner Shadows）**：进度条内槽等区域使用内阴影，表现出材质被向下雕刻镂空的立体感。

### 4.2 宠物主题与美术规范

提供多种不同美术风格的宠物图包供班级选择：

| 风格 | 标识 | 说明 |
|------|------|------|
| 可爱风 | Chibi / Kawaii | Q版萌系 |
| 像素风 | Pixel Art | 复古游戏像素风 |
| 写实风 | Realistic | 偏写实 3D 风格 |
| 玩偶风 | Jellycat | 毛绒玩偶质感 |

**切换规则**：

- 一个班级只能选择一种宠物风格
- 创建班级时选定风格
- 中途切换风格需要随机替换宠物形象，或让学生重新选择
- 教师可在「教师设置 → 宠物与主题配置」中一键切换

> 💡 宠物种类和风格主题可持续扩展，后续版本逐步增加新系列和节假日限定主题。

### 4.3 核心交互说明

#### 4.3.1 打分弹窗交互（Scoring Popup / 学生档案）

- **视觉风格**：采用侧边栏 + 主内容区的复合布局，延续 Claymorphism 立体卡片风格。所有交互按钮（如标签、关闭按钮）均采用仿多邻国（Duolingo Style）的 3D 厚重感设计，具有加粗描边与硬阴影。
- **布局结构**：
  - **Header**：左侧展示“学生档案”标题与副标题，右侧包含“分享”按钮与 3D 样式的“关闭”按钮。
  - **Sidebar（左侧）**：展示学生当前宠物的 3D 图像、神兽等级（Lv.1 ~ Lv.5）、名称（如“玄武”）、总能量（Total Energy）以及已获得的徽章数。
  - **Main Content（右侧）**：
    - **Tab 切换**：包含“喂养/惩罚”、“成长记录”、“神兽图鉴”三个标签页。采用仿多邻国（Duolingo Style）的 3D 厚重感设计，具有加粗描边与硬阴影。
    - **搜索功能**：提供行为搜索框，方便快速定位加/扣分项。
    - **神兽图鉴页**：
        - **动态进化预览**：顶部显示 5 张等比例缩放的进化状态卡片（Lv.1 ~ Lv.5），点击下方图鉴中的神兽时，上方动态切换显示该神兽的完整进化路径。
        - **已收藏画廊**：仅展示该学生已达成 Lv.5 满级收集的神兽。
        - **勋章印章卡片**：采用定制化卡片，移除进度条，放大形象展示，并覆盖红色的“已收藏”3D 视觉印章。
- **分享功能**：点击“分享”按钮可生成该学生的只读成长档案视图。

#### 4.3.2 进化转场动画

- 每次宠物进化时，播放炫酷的动画转场
- 随着等级的提高，转场效果逐级更酷炫
- 满级时出现庆祝动画 + 徽章获得动效

#### 4.3.3 宠物卡片 Slider 轮播交互

宠物卡片设计为**可左右滑动的 Carousel 轮播组件**，允许用户浏览同一只宠物的 5 个进化形态：

- **导航**：通过左右箭头引导滑动浏览
- **形态角标**：卡片顶部显示当前形态等级（如"第 3 形态"）
- **底部进度点（SVG 进化追踪器）**：直观显示宠物在进化树中的位置与解锁情况；未解锁阶段置灰，已解锁阶段高亮
- **状态限制**：
  - 在【宠物图鉴】中，可**自由滑动**预览所有 5 个形态（无论解锁与否）
  - 在【班级主页】中，**只能浏览到当前已解锁的形态**，未解锁形态被屏蔽

## 六、非功能需求

### 6.1 性能要求

| 指标 | 要求 |
|------|------|
| 页面加载 | 首屏 ≤ 2 秒 |
| 打分响应 | 操作后 ≤ 500ms 反馈 |
| 进化动画 | 流畅播放，无卡顿 |
| 并发支持 | 单班级 ≤ 100 学生流畅操作 |

### 6.2 数据安全

- 所有数据云端存储，换设备登录同一账号可查看全部数据
- 删除班级需二次确认（输入班级名称）
- 打分操作支持撤回
- 激活码一码一用，防盗刷

### 6.3 设备适配

- 响应式设计，适配电脑 / 平板 / 手机 / 希沃一体机
- 支持 PWA 安装到桌面
- 手机端全功能可用
